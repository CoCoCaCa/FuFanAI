## 什么是扶帆AI

**不是**用 AI 模拟一家公司怎么运转（配置成本高）。

**而是**：你告诉 AI 要完成什么，AI 自己组织数字员工起来干活。

```text
你："帮我开发一个外卖App"
    ↓
管理型Agent（项目负责人）自动调度：
    产品设计 → 开发实现 → 测试验证 → 上线部署
    ↓
你只需要审批关键节点，其余全部自动执行
```

---

## 核心概念

### 数字员工（Agent）

| 类型 | 角色 | 职责 |
|------|------|------|
| **专家型Agent** | 执行具体任务 | 专注专业能力，如：产品经理、架构师、开发、测试、运维等 |
| **管理型Agent** | 项目负责人 | 解析任务、调度专家Agent、汇报进度 |

### 技能（Skill）—— 双层结构

```text
管理型Agent Skill（流程层）
    │
    ├── 步骤1：产品设计 → 产品专家Agent
    ├── 步骤2：UI设计 → UI专家Agent
    ├── 步骤3：人工审核（暂停，等你确认）
    ├── 步骤4：开发实现 → 开发专家Agent
    └── ...

专家型Agent Skill（执行层）
    └── 知道需要什么输入、产出什么、自己执行
```

**关键设计**：管理管流程，专家管执行，职责边界清晰。

### 信息流转

```text
你的对话（短期）
    ↓
项目工作区（共享，所有Agent可读写）
    ↓
Agent私有记忆（持久，每个Agent独立）
```

## 快速开始

### 初始化配置

1. **双击 exe** → 选择工作目录
2. **配置 nanobot 服务** → 配置大模型参数 → 点击保存
    > 本项目基于 [Nanobot](https://github.com/HKUDS/nanobot) 开源框架二次开发构建，详细配置可前往开源项目查看与学习，感谢开源社区的支持。
3. **重启 nanobot 服务** → 验证：工作目录下是否生成 `workspace` 文件夹
4. **创建员工** → 员工管理 → 添加数字员工
5. **创建项目** → 项目列表 → 新建项目

### 常用操作

| 操作 | 说明 |
|------|------|
| 项目群聊 | 创建项目后进入群聊，对话默认发送至项目agent，指定发送agent使用@ |
| 单聊 | 员工管理 → 点击员工 → 一对一沟通 |
| 定时任务 | 在项目群里中@agent 或 默认项目agent 进行创建定时任务 |
| 聊天引用 | 长按消息 → 引用 → 精准回复 |

## 安全与隐私

- 所有数据存储在本地工作目录，服务器不收集用户数据
- API 密钥仅存储在本地配置中，不会上传至任何服务器
- 支持离线本地使用（部分功能需要联网）

## 为什么选择扶帆AI

| 对比维度 | 现有AI工具 | 扶帆AI |
|----------|------------|--------|
| 记忆能力 | 无持久记忆，每会话独立 | 真正的持久记忆 |
| 智能体架构 | 单全能智能体 | 多智能体协作，分工明确 |
| 操作难度 | 复杂prompt调试，高运维 | 操作便利，零配置上手 |
| 成本控制 | token消耗大 | 专为成本控制设计 |
| 流程透明 | 黑盒不可知 | 全流程透明可审计 |

**核心优势**：
- 多智能体协作：多个数字员工各司其职，协作完成复杂任务
- 真正持久记忆：每次对话都记得，不丢失上下文
- 零配置上手：说句话就能用，不需要写复杂的prompt
- 本地优先：数据存储在本地，安全可控

## 常见问题

**Q: 数据存储在哪里？**
A: 所有数据存储在您本地的工作目录，服务器不收集任何数据。

**Q: 支持哪些大模型？**
A: 支持 OpenAI API 兼容的所有模型，配置路径：设置 → 大模型参数。

**Q: 如何创建自定义 Agent？**
A: 员工管理 → 添加数字员工 → 选择角色类型和技能。

**Q: 工作目录可以迁移吗？**
A: 可以，直接迁移整个工作目录文件夹即可。
